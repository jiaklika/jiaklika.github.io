0-环境部署

思考并回答以下问题：

调试

使用VS Code进行调试，来看一个简单的返回大值的程序，在main.go文件输入如下内容。

  • os.Args从命令行获取参数列表，第一个参数通常是程序文件本身，可以忽略。
  • strconv.Atoi把字符串转为整型（这个程序只是做简单的演示，没有进一步判断字符串是否可以转为整型）。

创建调试文件，ctrl+shift+D，点击创建launch.json文件。

因为我们的参数是从命令行获取，所以在调试时需要提前设置好命令行参数，打开launch.json文件，在program下面新增args字段。

然后切换到main.go标签，在11行和19行打断点，按下F5进行调试，进行单步调试，观察左侧变量区、监视区、调用堆栈区的变化。

单元测试

只有一个经历了完善单元测试的函数或功能，才能被安心地用于生产环境。

还是用上面的求大值示例来进行演示，VS Code提供了非常强大的工具，可以一键生成单元测试的大部分代码，右击函数体区域(下图红框区域)，点击Go: Generate Unit Tests For Function。

会自动生成并打开main_test.go文件，测试代码已经组织好，我们只需要编写测试用例，填充结构体数组就可以了。

  • args是被测函数的参数结构体。
  • tests是所有的单测用例，是个匿名结构体。
    • name是该单测名称。
    • args是输入参数。
    • want是期望输出。
  • int(^uint(0)>>1)代表的是整型的最大值。

编写好单元测试后，点击测试函数上方的run test。

运行如果没问题，都符合预期输出，那会显示ok的字眼，否则就排查一下函数编写是否正确，或者单测设计是否合理，如有必要，还要进行debug test。

实验总结

本次课程我们介绍了如何调试程序。在编写好功能后，如何进行单元测试保证功能的可用性。读者朋友可以把上面某个单元测试的期望故意改错，观察一下测试结果有什么变化。
