Beego 控制器
在 Web 开发中，控制器的职责就是根据请求输出响应。

熟悉 Go 语法的读者朋友们应该能清楚地知道，这是结构体的组合，类似于面向对象语言中的继承。

虽然以继承来类比组合，但本质上继承和组合是不同的东西，是 is a 和 has a 的区别。

再来看看 beego.Controller 结构体有哪些元素，分别有什么用。

上下文信息
Ctx 主要用于存储上下文信息。
Data 存储上下文数据。
路由信息
controllerName 首字母小写，只对包内可见，可以理解为私有变量，控制器名称。
actionName 方法名称。
methodMapping 通过哪种 HTTP 方式访问，Get、Post 等。
AppController Beego 应用。
模板信息
TplName 模板名称。
ViewPath 视图层路径。
Layout 模板通用结构，如后台管理系统，只有主体内容部分是经常变换，菜单和头部分基本不变，就可以定义 Layout。
LayoutSection 模板通用结构子段，还是后台管理系统，如菜单部分就可以定义为一个 Layout Section。
xsrf 数据（xsrf 可以跨站请求伪造，确保数据资产安全）
_xsrfToken 访问凭证。
XSRFExpire 凭证到期时间。
EnableXSRF 是否启用。
session 相关
CruSession 当前会话信息。
上下文、请求和响应
上下文中存储了当前请求的数据及响应，是控制器中的重点。

你可以使用 Request 和 ResponseWriter 去直接操作请求和响应，也可以通过 Input 和 Output 的 Api 去操作，会更加方便，因为它是对前者的封装。

为了佐证这一点，我们节选一些代码片断来分析，在 Beego 源码中的 input.go 和 output.go，快捷键 Ctrl+单击 可以快速跳转到类型定义的地方。

Protocol 方法，返回当前请求使用的协议。
URI 方法，返回当前请求的路径，请求参数及路径 hash。
URL 方法，返回当前请求的路径，不包括请求参数及路径 hash。

Header 方法，设置当前响应的头信息。
